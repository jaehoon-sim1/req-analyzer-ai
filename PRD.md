# PRD: 요구사항 분석 AI

> **Version:** 1.0
> **작성일:** 2026-03-13
> **상태:** Draft

---

## 1. 기능 목적

요구사항의 모호함이나 누락으로 인해 발생하는 버그 및 커뮤니케이션 비용을 줄이고,
QA 단계 이전에 요구사항 품질을 개선한다.

---

## 2. 목표

사용자가 요구사항 문서 또는 텍스트를 입력하면 AI가 자동으로 분석하여 다음 정보를 제공한다.

- 요구사항 요약
- 기능 목록 정리
- 주요 테스트 포인트
- 모호한 요구사항 탐지
- 누락 가능 요구사항 탐지
- QA 질문 리스트 생성

---

## 3. 사용자 유형

| 사용자 | 설명 |
|--------|------|
| QA 엔지니어 | 요구사항을 검토하고 테스트 전략을 수립하는 주 사용자 |

---

## 4. 입력(Input)

사용자는 아래 방식으로 요구사항을 입력할 수 있다.

### 4-1. 텍스트 입력

요구사항 문서를 텍스트로 직접 입력한다.

### 4-2. 문서 업로드

지원 파일 형식:

| 파일 형식 | 비고 |
|-----------|------|
| PDF | 텍스트 기반 PDF 파싱 |
| DOCX | Microsoft Word 문서 파싱 |
| PNG | OCR을 통한 텍스트 추출 |
| JPG | OCR을 통한 텍스트 추출 |
| TXT | 텍스트 파일 직접 읽기 |

> **[추가] 입력 제약 조건**
>
> - 텍스트 입력: 최대 입력 글자 수 정의 필요 (예: 50,000자)
> - 파일 업로드: 최대 파일 크기 정의 필요 (예: 10MB)
> - 이미지(PNG/JPG)의 경우 OCR 정확도 한계가 존재하므로, 텍스트 기반 입력을 권장하는 안내 필요

---

## 5. 기능 요구사항

### FR-001 요구사항 분석

사용자가 입력한 요구사항 문서를 분석해야 한다.

### FR-002 요구사항 요약 생성

AI는 요구사항을 요약하여 핵심 기능을 정리해야 한다.

**출력 예시:**

```
요약:
사용자는 이메일과 비밀번호를 이용해 로그인할 수 있다.
```

### FR-003 기능 목록 생성

요구사항을 기반으로 기능 리스트를 생성한다.

**출력 예시:**

| # | 기능 목록 |
|---|-----------|
| 1 | 로그인 기능 |
| 2 | 이메일 validation |
| 3 | 비밀번호 validation |
| 4 | 오류 메시지 표시 |

### FR-004 테스트 포인트 생성

요구사항을 기반으로 QA 테스트 포인트를 생성한다.

**출력 예시:**

- 입력 validation
- 로그인 성공
- 로그인 실패
- 오류 메시지 표시

### FR-005 모호한 요구사항 탐지

AI는 요구사항 중 명확하지 않은 부분을 식별해야 한다.

**출력 예시:**

- 로그인 실패 횟수 제한이 정의되지 않음
- 비밀번호 정책이 정의되지 않음

### FR-006 누락 가능 요구사항 탐지

> **[추가] 기존 목표에는 포함되어 있으나 FR이 정의되지 않았으므로 추가**

AI는 입력된 요구사항에서 일반적으로 포함되어야 하나 누락된 항목을 탐지한다.

**출력 예시:**

- 에러 핸들링 시나리오 누락
- 세션 만료 처리 누락
- 접근 권한 정의 누락

### FR-007 QA 질문 생성

요구사항 검증을 위한 QA 질문 리스트를 생성한다.

**출력 예시:**

| # | QA 질문 |
|---|---------|
| 1 | 로그인 실패 횟수 제한이 있나요? |
| 2 | MFA 지원 예정인가요? |
| 3 | 비밀번호 정책은 무엇인가요? |

### FR-008 문서 파싱

> **[추가] 파일 업로드 입력을 처리하기 위한 파싱 기능 정의 필요**

업로드된 문서(PDF, DOCX, PNG, JPG, TXT)에서 텍스트를 추출한다.

- PDF: 텍스트 레이어 추출
- DOCX: 본문 텍스트 추출
- PNG/JPG: OCR을 통한 텍스트 추출
- TXT: 직접 읽기

파싱 실패 시 사용자에게 오류 메시지를 표시한다.

### FR-009 결과 내보내기

> **[추가] NFR-003에서 Excel/JSON 출력을 요구하므로, 이를 기능 요구사항으로 명시**

분석 결과를 다음 형식으로 내보낼 수 있다.

- Excel (.xlsx)
- JSON (.json)

## 7. 비기능 요구사항

### NFR-001 응답 속도

- 요구사항 분석 결과는 **180초 이내**에 생성되어야 한다.
- 분석 진행 상황을 사용자에게 시각적으로 표시한다.
  - 예: 진행현황 로딩바, 현재 처리 단계 표시

### NFR-002 사용 편의성

- 사용자는 **3단계 이내**로 분석 결과를 확인할 수 있어야 한다.
  - Step 1: 요구사항 입력 (텍스트 입력 또는 파일 업로드)
  - Step 2: 분석 실행 (버튼 클릭)
  - Step 3: 결과 확인

### NFR-003 결과 구조화

AI 결과는 구조화된 포맷으로 제공되어야 한다.

- Excel (.xlsx)
- JSON (.json)

### NFR-004 보안 및 데이터 처리

> **[추가] 업로드된 문서에 민감 정보가 포함될 수 있으므로 정의 필요**

- 업로드된 파일 및 입력 데이터는 분석 완료 후 서버에 보관하지 않는다. (또는 보관 정책 정의)
- 사용자 데이터는 외부로 전송되지 않는다. (또는 AI API 호출 시 데이터 처리 범위 정의)

### NFR-005 에러 처리

> **[추가] 다양한 실패 시나리오에 대한 처리 필요**

- 파일 파싱 실패 시 사용자에게 명확한 오류 메시지를 표시한다.
- AI 분석 실패/타임아웃 시 재시도 안내 또는 오류 메시지를 표시한다.
- 지원하지 않는 파일 형식 업로드 시 안내 메시지를 표시한다.

## 9. Acceptance Criteria

### AC-001 요구사항 분석 성공

- **Given** 사용자가 요구사항 문서를 입력한다
- **When** AI가 분석을 수행한다
- **Then** 요구사항 요약과 기능 목록이 생성된다

### AC-002 QA 질문 생성

- **Given** 요구사항 문서가 입력된다
- **When** AI가 분석을 수행한다
- **Then** QA 질문 리스트가 생성된다

### AC-003 모호한 요구사항 탐지

> **[추가] FR-005에 대한 AC 누락**

- **Given** 모호한 표현이 포함된 요구사항이 입력된다
- **When** AI가 분석을 수행한다
- **Then** 모호한 요구사항 목록과 개선 제안이 표시된다

### AC-004 누락 가능 요구사항 탐지

> **[추가] FR-006에 대한 AC 누락**

- **Given** 요구사항 문서가 입력된다
- **When** AI가 분석을 수행한다
- **Then** 일반적으로 필요하나 누락된 요구사항 목록이 표시된다

### AC-005 파일 업로드 분석

> **[추가] 파일 업로드 시나리오에 대한 AC 누락**

- **Given** 사용자가 지원 형식(PDF/DOCX/PNG/JPG/TXT)의 파일을 업로드한다
- **When** 시스템이 파일을 파싱하고 AI가 분석을 수행한다
- **Then** 텍스트 입력과 동일한 분석 결과가 생성된다

### AC-006 결과 내보내기

> **[추가] NFR-003 / FR-009에 대한 AC 누락**

- **Given** 분석 결과가 생성되어 표시된다
- **When** 사용자가 Excel 또는 JSON 내보내기를 선택한다
- **Then** 해당 형식의 파일이 다운로드된다

### AC-007 응답 시간

> **[추가] NFR-001에 대한 AC 누락**

- **Given** 사용자가 요구사항을 입력하고 분석을 실행한다
- **When** AI가 분석을 수행한다
- **Then** 180초 이내에 결과가 표시되고, 분석 중 진행 상황이 로딩바로 표시된다

### AC-008 에러 처리

> **[추가] NFR-005에 대한 AC 누락**

- **Given** 사용자가 지원하지 않는 파일 형식을 업로드한다
- **When** 시스템이 파일을 검증한다
- **Then** "지원하지 않는 파일 형식입니다" 오류 메시지가 표시된다

## 10. Scope 정의

> **[추가] 범위 경계를 명확히 하여 스코프 크리프 방지**

### In Scope

- 요구사항 텍스트 및 문서 기반 분석
- 단일 문서 단위 분석
- 한국어/영어 요구사항 분석

### Out of Scope

- 요구사항 버전 관리 및 변경 이력 추적
- 다수 문서 간 요구사항 교차 분석
- Jira, Confluence 등 외부 도구 연동
- 테스트케이스 자동 생성 (별도 기능으로 분리)
