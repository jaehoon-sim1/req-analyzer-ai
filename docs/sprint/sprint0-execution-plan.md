## 태스크 크기 정의 ("30분 규칙")

> **하나의 Sprint 태스크는 "30분 안에 결과를 눈으로 확인할 수 있는 크기"로 정의한다.**

### 원칙

| 기준 | 설명 |
|------|------|
| **시간 상한** | 하나의 태스크는 최대 30분 이내에 완료 가능해야 한다 |
| **시각적 검증** | 완료 시 화면, 로그, 테스트 결과 등으로 "눈으로 확인"할 수 있어야 한다 |
| **독립 실행** | 다른 태스크 완료를 기다리지 않고 단독으로 실행·검증 가능해야 한다 |
| **분해 기준** | 30분을 초과하면 2~3개 하위 태스크로 분해한다 |

### 검증 방법 예시

| 태스크 유형 | 30분 내 눈으로 확인하는 방법 |
|-------------|------------------------------|
| API 연동 | curl/Postman으로 호출 → JSON 응답 확인 |
| 프롬프트 작성 | 샘플 입력 → AI 출력 → 기대 결과 비교 |
| UI 컴포넌트 | 브라우저에서 렌더링 → 시각적 확인 |
| 파서 구현 | 단위 테스트 실행 → 통과/실패 확인 |
| 문서 작성 | 문서 열어서 내용 확인 |

### 적용 예시

```
❌ 나쁜 예: "AI 분석 파이프라인 구축" (2시간 소요, 범위 모호)
✅ 좋은 예:
  ├── (15min) Claude API 호출 → 텍스트 응답 수신 확인
  ├── (15min) JSON 파싱 로직 → 단위 테스트 통과
  └── (15min) 6개 섹션 순차 호출 → 전체 JSON 결과 확인
```
